# Contributing Giudelines
We are very open to Contributions, but there are a few Rules.
- We only accept Contributions over Pull Requests
- We only accept contributions that add a new Feature that fit into the already existing Website or Roadmap (If you want to suggest smth completely new, please tell us about it in the Discussions) or Fix an existing Bug.
- If the Website isnt fully made yet, please use the backend files that are already in the Server Folder to help connect with the Backend and keeo everything in the Frontend as close as possible to the Design that will released soon. (It will be linked here and in other Placees)
- Enable "allow edits by maintainer" in every Pull request, so if we find a Bug or smth, we can correct it ourselves. (If the Bug is too big tho, we will request a change.)
- Please limit yourself with the usage of AI, because if its crap code we wont accept it :)
- Dont make massive Pull requests. Please create a Pull request after every major change you make, i dont want multiple different Fetures in one Pull Request. If it really is a new Feature, that is very big and takes much lines, please describe it in the describtion of the Pull request, because if we dont understand that its really a important change we wont just accept it, cause we have to read through the entire code ourselves.
